<task type="auto">
  <name>Task 2: Create Session and Piece Prisma models</name>
  <files>prisma/schema.prisma</files>
  <action>
Add the Session and Piece models to Prisma schema. Session replaces the Practice/Workout concept with a cleaner hierarchy.

Session model:
- id: String @id @default(cuid())
- name: String
- type: SessionType enum (ERG, ROW, LIFT, RUN, CROSS_TRAIN, RECOVERY)
- date: DateTime
- startTime: DateTime?
- endTime: DateTime?
- recurrenceRule: String? (RRULE format)
- notes: String? @db.Text
- status: SessionStatus enum (PLANNED, ACTIVE, COMPLETED, CANCELLED)
- teamId: String (relation to Team)
- createdById: String (relation to User)
- createdAt: DateTime @default(now())
- updatedAt: DateTime @updatedAt
- pieces: Piece[]
- sessionCode: String? @unique (for live joining)
- athleteVisibility: Boolean @default(true) (can athletes see each other's data)

Piece model:
- id: String @id @default(cuid())
- sessionId: String (relation to Session)
- segment: PieceSegment enum (WARMUP, MAIN, COOLDOWN)
- name: String (e.g., "4x2000m", "40' SS")
- description: String?
- order: Int
- distance: Int? (meters)
- duration: Int? (seconds)
- targetSplit: Int? (seconds per 500m)
- targetRate: Int? (strokes per minute)
- targetWatts: Int?
- targetHRZone: String?
- targetRPE: Int?
- notes: String?
- boatClass: String? (for ROW sessions)
- sets: Int? (for LIFT sessions)
- reps: Int? (for LIFT sessions)

SessionType enum: ERG, ROW, LIFT, RUN, CROSS_TRAIN, RECOVERY
SessionStatus enum: PLANNED, ACTIVE, COMPLETED, CANCELLED
PieceSegment enum: WARMUP, MAIN, COOLDOWN

Add indexes:
- Session: @@index([teamId, date])
- Piece: @@index([sessionId, order])
  </action>
  <verify>npx prisma validate</verify>
  <done>Prisma schema validates with Session and Piece models</done>
</task>
