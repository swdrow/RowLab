Create TanStack Query hooks for erg test operations following useAthletes.ts pattern:

1. **useErgTests(filters?: ErgTestFilters)** - Main hook:
   - Query: GET /api/v1/erg-tests with query params from filters
   - Mutations: create, update, delete with optimistic updates
   - Returns: tests, isLoading, error, createTest, updateTest, deleteTest, etc.
   - Query key: ['ergTests', filters] (include filters in key for proper caching)
   - staleTime: 2 minutes

2. **useAthleteErgHistory(athleteId: string)** - For athlete profile:
   - Query: GET /api/v1/erg-tests/athlete/:athleteId/history
   - Returns: tests, personalBests, totalTests, isLoading, error
   - Query key: ['ergTests', 'athlete', athleteId]
   - enabled: !!athleteId

3. **useErgLeaderboard(testType: TestType, limit?: number)** - For rankings:
   - Query: GET /api/v1/erg-tests/leaderboard?testType=...&limit=...
   - Returns: leaderboard, isLoading, error
   - Query key: ['ergTests', 'leaderboard', testType, limit]

4. **useBulkImportErgTests()** - For CSV import:
   - Mutation: POST /api/v1/erg-tests/bulk-import
   - Invalidates ['ergTests'] on success
   - Returns: importTests, isImporting, importError, importResult

Use auth store check for enabled flag (isAuthenticated && isInitialized).
Format API response data to match types.

<task type="auto">
  <name>Task 3: Create useConcept2 hooks</name>
  <files>src/v2/hooks/useConcept2.ts</files>
  <action>
Create TanStack Query hooks for Concept2 status and sync:

1. **useConcept2Status(athleteId?: string)** - Get connection status:
   - If athleteId: GET /api/v1/concept2/status/:athleteId
   - If no athleteId (for current user): GET /api/v1/concept2/status/me
   - Returns: status (C2Status), isLoading, error, isConnected (derived boolean)
   - Query key: ['concept2', 'status', athleteId || 'me']
   - staleTime: 5 minutes (status doesn't change frequently)

2. **useTriggerC2Sync(athleteId?: string)** - Trigger manual sync:
   - If athleteId: POST /api/v1/concept2/sync/:athleteId
   - If no athleteId: POST /api/v1/concept2/sync/me
   - Invalidates: ['ergTests'], ['concept2', 'status']
   - Returns: triggerSync, isSyncing, syncError, syncResult

3. **useTeamC2Statuses()** - Get C2 status for all athletes (for overview):
   - This is derived: fetch athletes, then for each with concept2UserId, query status
   - OR: Create a new array that maps athleteId -> C2Status
   - Note: Backend doesn't have a bulk status endpoint, so we'll do client-side aggregation
   - Return: Map<athleteId, C2Status>, isLoading

Implementation note: For useTeamC2Statuses, since there's no bulk endpoint, use useAthletes() to get roster, filter for athletes with concept2UserId, and render individual status indicators. Don't pre-fetch all statuses - let individual components query as needed to avoid N+1 at load time.
