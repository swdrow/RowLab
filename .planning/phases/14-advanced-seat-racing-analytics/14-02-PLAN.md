<feature>
  <name>Bradley-Terry Model Implementation with Boat Speed Normalization</name>
  <files>server/services/bradleyTerryService.js, server/services/__tests__/bradleyTerryService.test.js</files>
  <behavior>
The Bradley-Terry model estimates relative strengths from pairwise comparisons:
  P(i beats j) = pi_i / (pi_i + pi_j)

Extended model with boat speed bias correction:
  P(i in boat_a beats j in boat_b) = (pi_i * gamma_a) / (pi_i * gamma_a + pi_j * gamma_b)

Where gamma_a is the boat speed parameter for shell a (gamma > 1 means faster shell).

**Test Cases:**
1. Simple 2-athlete case: A beats B 3 times, B beats A 1 time
   - A's strength should be higher than B's
   - Probability P(A beats B) should be ~0.75

2. 3-athlete case: A > B > C (transitive)
   - Rankings should be A, B, C in order
   - P(A beats C) > P(A beats B)

3. Sparse data: A vs B only, B vs C only
   - Should still produce rankings via transitivity
   - A vs C should have lower confidence

4. Edge case: All ties (everyone beats everyone equally)
   - All strengths should be approximately equal
   - All probabilities ~0.5

5. Standard errors increase with fewer comparisons

6. **Boat speed bias test:** A always in fast shell, B always in slow shell
   - Without correction: A appears stronger
   - With correction: Shell speed parameter captures difference, A and B more equal

7. **Shell usage tracking test:** Verify shell parameters are computed
   - shells: [{shellId, speedParameter, comparisonsInShell}]
  </behavior>
  <implementation>
Use fmin library for Nelder-Mead optimization of negative log-likelihood.

Key functions:
- fitBradleyTerryModel(comparisons, options) → { athletes: [{athleteId, strength, stdError, ci}], shells: [{shellId, speedParam}], convergence }
- computeProbabilityMatrix(strengths) → P[i][j] matrix
- computeStandardErrors(hessian, solution) → standard errors via Fisher information

Options:
- includeBoatSpeed: boolean (default true) - Whether to estimate shell speed parameters
- normalizeShells: boolean (default true) - Constrain geometric mean of shell params to 1

Boat Speed Normalization Algorithm:
1. Extract unique shell IDs from comparisons
2. If includeBoatSpeed is true, estimate both athlete strengths AND shell speed parameters
3. Joint optimization: maximize likelihood over (athlete params, shell params)
4. Identifiability constraint for athletes: Sum of log-strengths = 0
5. Identifiability constraint for shells: Product of gamma = 1 (geometric mean = 1)

Handle edge cases:
- Empty comparisons → error
- Single athlete → return default
- Disconnected graph → warning + fit connected components separately
- No shell data → skip boat speed estimation, fall back to basic model
  </implementation>
</feature>
