Create server/services/availabilityService.js with three functions:

1. `getTeamAvailability(teamId, { startDate, endDate })`:
   - Get all athletes in team via Athlete model (where teamId matches)
   - Get availability records for date range
   - Return structured data: array of { athleteId, athleteName, dates: [{ date, morningSlot, eveningSlot }] }

2. `getAthleteAvailability(athleteId, { startDate, endDate })`:
   - Get availability for single athlete within date range
   - Return array of { date, morningSlot, eveningSlot, notes }
   - Order by date ascending

3. `updateAthleteAvailability(athleteId, availabilityArray)`:
   - Input: array of { date, morningSlot, eveningSlot, notes? }
   - Use Prisma transaction to upsert each date (by athleteId+date unique key)
   - Return updated availability records

Use Prisma schema models:
```prisma
model Availability {
  id           String           @id @default(uuid())
  athleteId    String
  date         DateTime
  morningSlot  AvailabilitySlot @default(NOT_SET)
  eveningSlot  AvailabilitySlot @default(NOT_SET)
  notes        String?

  @@unique([athleteId, date])
}

enum AvailabilitySlot {
  AVAILABLE
  UNAVAILABLE
  MAYBE
  NOT_SET
}
```

<task type="auto">
  <name>Task 2: Create availability routes and mount</name>
  <files>server/routes/availability.js, server/index.js</files>
  <action>
Create server/routes/availability.js with endpoints:

**GET /api/v1/availability/team**
- Middleware: authenticateToken, teamIsolation
- Query params: startDate (required), endDate (required) - both ISO8601 dates
- Validation:
  - query('startDate').isISO8601()
  - query('endDate').isISO8601()
- Returns { success: true, data: { availability: [...] } } with team-wide grid data

**GET /api/v1/availability/:athleteId**
- Middleware: authenticateToken, teamIsolation
- Validation: param('athleteId').isUUID(), query startDate/endDate
- Verify athlete belongs to user's team before returning
- Returns athlete's availability for date range

**PUT /api/v1/availability/:athleteId**
- Middleware: authenticateToken, teamIsolation
- Authorization check: User can only edit their own availability (user.athleteId === athleteId) OR user is COACH/OWNER
- Validation:
  - param('athleteId').isUUID()
  - body('availability').isArray()
  - body('availability.*.date').isISO8601()
  - body('availability.*.morningSlot').isIn(['AVAILABLE', 'UNAVAILABLE', 'MAYBE', 'NOT_SET'])
  - body('availability.*.eveningSlot').isIn(['AVAILABLE', 'UNAVAILABLE', 'MAYBE', 'NOT_SET'])
  - body('availability.*.notes').optional().isString().isLength({ max: 200 })
- Returns updated availability

Authorization logic for PUT:
```javascript
// Athletes can only edit their own availability
// Coaches/owners can edit any athlete's availability
const canEdit = req.user.athleteId === req.params.athleteId ||
                ['COACH', 'OWNER'].includes(req.user.activeTeamRole);
if (!canEdit) {
  return res.status(403).json({
    success: false,
    error: { code: 'FORBIDDEN', message: 'Cannot edit other athletes\' availability' }
  });
}
```

Mount in server/index.js:
```javascript
import availabilityRoutes from './routes/availability.js';
app.use('/api/v1/availability', apiLimiter, availabilityRoutes);
```
