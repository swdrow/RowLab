  <files>src/v2/components/lineup/LineupWorkspace.tsx</files>
  <action>
Add responsive breakpoint to switch between desktop and mobile layouts:

1. Create useIsMobile hook or use media query:
   ```typescript
   const isMobile = useMediaQuery('(max-width: 768px)');
   // Or simpler:
   const [isMobile, setIsMobile] = useState(false);
   useEffect(() => {
     const checkMobile = () => setIsMobile(window.innerWidth < 768);
     checkMobile();
     window.addEventListener('resize', checkMobile);
     return () => window.removeEventListener('resize', checkMobile);
   }, []);
   ```

2. Conditional rendering:
   ```tsx
   return isMobile ? (
     <MobileLineupBuilder />
   ) : (
     // Existing desktop layout with DndContext
     <DndContext ...>
       ...
     </DndContext>
   );
   ```

3. Shared state:
   - Both layouts use same lineupStore
   - Changes in mobile reflect in desktop and vice versa
   - Undo/redo works across both

4. DndContext only on desktop:
   - Mobile doesn't need drag-drop context
   - Simpler, avoids touch conflicts

5. Test scenarios:
   - Start on mobile, rotate to landscape
   - Start on desktop, resize to mobile width
   - State persistence across switches
  </action>
