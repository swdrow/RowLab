<task type="auto">
  <name>Task 2: Create Theme Hook with System Preference</name>
  <files>src/v2/hooks/useTheme.ts</files>
  <action>
Create the useTheme hook that manages theme state with system preference detection and localStorage persistence.

**Implementation:**
- Define `Theme` type as `'dark' | 'light' | 'field'`
- Use useState for `theme` with initializer that:
  1. Checks localStorage('v2-theme') first (user override)
  2. Falls back to system preference via `window.matchMedia('(prefers-color-scheme: dark)')`
  3. Default to 'dark' if no system preference
- Track `isSystemDefault` boolean (true when no localStorage override)
- useEffect to listen for system preference changes via `mediaQuery.addEventListener('change', ...)`
- Only update theme on system change if `isSystemDefault` is true
- `setTheme` function: updates state, writes to localStorage, sets isSystemDefault to false
- `clearThemePreference` function: removes from localStorage, sets isSystemDefault to true, applies system preference
- Return: `{ theme, setTheme, isSystemDefault, clearThemePreference }`

Follow the pattern from 02-RESEARCH.md Pattern 2 exactly. This replaces the inline theme logic currently in V2Layout.tsx.
  </action>
  <verify>
TypeScript compiles: `npx tsc --noEmit src/v2/hooks/useTheme.ts`
  </verify>
  <done>
- useTheme hook returns { theme, setTheme, isSystemDefault, clearThemePreference }
- Theme respects system preference when no localStorage override
- Theme persists to localStorage on manual change
- System preference changes update theme only when isSystemDefault is true
  </done>

<task type="auto">
  <name>Task 3: Create Shared Stores Hook for V1/V2 Integration</name>
  <files>src/v2/hooks/useSharedStores.ts</files>
  <action>
Create the shared stores integration layer that allows V2 components to access V1 Zustand stores.

**Implementation:**
- Import `createContext`, `useContext` from 'react'
- Import `useAuthStore` from '@/store/authStore'
- Import `useSettingsStore` from '@/store/settingsStore'
- Create `AuthStoreContext = createContext(useAuthStore)` - share store INSTANCE, not values
- Create `SettingsStoreContext = createContext(useSettingsStore)` - share store INSTANCE, not values
- Export `useV2Auth = () => useContext(AuthStoreContext)` - returns the store hook itself
- Export `useV2Settings = () => useContext(SettingsStoreContext)` - returns the store hook itself
- Export the Context objects for use in providers (ShellLayout will wrap with these)

**Critical:** Share store *instances* via Context, NOT store values. The pattern is:
```typescript
// CORRECT - shares instance
const AuthStoreContext = createContext(useAuthStore);
// WRONG - shares current values, causes re-render loops
const AuthStoreContext = createContext(useAuthStore());
```

This follows 02-RESEARCH.md Pattern 1 and avoids Pitfall 3 (re-render loops).
  </action>
