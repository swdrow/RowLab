  <action>
Create the WorkspaceSidebar component following the pattern from 02-RESEARCH.md "Workspace Sidebar with Context-Aware Navigation" code example.

**Structure:**
```
<nav aria-label="[context] workspace navigation">
  <Link to="/beta/..." />  {/* Repeated for each nav item */}
</nav>
```

**Navigation items by context (from contextStore):**

Me context:
- Dashboard → /beta/dashboard (HomeIcon)
- My Workouts → /beta/workouts (ActivityIcon)
- Progress → /beta/progress (TrendingUpIcon)

Coach context:
- Athletes → /beta/athletes (UsersIcon)
- Training Plans → /beta/plans (CalendarIcon)
- Lineups → /beta/lineups (BoatIcon)

Admin context:
- Users → /beta/users (UsersIcon)
- Teams → /beta/teams (TeamIcon)
- Settings → /beta/settings (SettingsIcon)

**Implementation details:**

1. **Get navigation from context store:**
   - Import `useContextStore` from '@v2/stores/contextStore'
   - The store should provide `getNavItems()` or `navItemsByContext[activeContext]`
   - If store doesn't have nav items yet, define them locally using the same pattern

2. **Active route detection:**
   - Import `useLocation` from 'react-router-dom'
   - Compare `location.pathname` with each nav item's `to` property
   - Apply active styles when matched

3. **Icons:** Use inline SVG icons (Lucide-style) matching the context:
   - HomeIcon, ActivityIcon, TrendingUpIcon for Me
   - UsersIcon, CalendarIcon, custom BoatIcon for Coach
   - UsersIcon, TeamIcon, SettingsIcon for Admin

4. **Styling:**
   - Sidebar width: `w-64` (256px)
   - Full height: `h-full`
   - Background: `bg-bg-surface`
   - Border right: `border-r border-border-default`
   - Nav items: flex row, gap-3 between icon and text, px-3 py-2, rounded-lg
   - Inactive: `text-text-secondary hover:bg-bg-hover hover:text-text-primary`
   - Active: `bg-action-primary text-button-primary-text` (from design tokens)
   - Transition: `transition-colors duration-150`

5. **Accessibility:**
   - `aria-label` on nav: "[Context] workspace navigation" (dynamic based on activeContext)
   - `aria-current="page"` on active link
   - Links are keyboard focusable (native Link behavior)

6. **Component structure:**
   ```tsx
   export function WorkspaceSidebar() {
     const { activeContext } = useContextStore();
     const location = useLocation();
     const navItems = getNavItemsForContext(activeContext);

     return (
       <nav aria-label={`${activeContext} workspace navigation`}>
         {navItems.map(item => (
           <Link
             key={item.to}
             to={item.to}
             aria-current={location.pathname === item.to ? 'page' : undefined}
             className={cn(
               'flex items-center gap-3 px-3 py-2 rounded-lg transition-colors duration-150',
               location.pathname === item.to
                 ? 'bg-action-primary text-button-primary-text'
                 : 'text-text-secondary hover:bg-bg-hover hover:text-text-primary'
             )}
           >
             <item.icon className="w-5 h-5" />
             <span className="font-medium">{item.label}</span>
           </Link>
         ))}
       </nav>
     );
   }
   ```

Use a simple `cn` utility for className merging (can be defined inline or use clsx if available).
