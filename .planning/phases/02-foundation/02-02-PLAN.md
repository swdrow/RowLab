  <action>
Create the ContextRail component following the pattern from 02-RESEARCH.md "Context Rail Component Structure" code example.

**Structure:**
```
<nav aria-label="Workspace contexts">
  <button> Me (icon + active indicator) </button>
  <button> Coach (icon + active indicator) </button>
  <button> Admin (icon + active indicator) </button>
</nav>
```

**Implementation details:**

1. **Icons:** Use inline SVG icons (Lucide-style) for each context:
   - Me: User icon (person silhouette)
   - Coach: Users icon (multiple people)
   - Admin: Settings/cog icon

2. **Active indicator:** Use Framer Motion `layoutId="activeContext"` on a background div that animates between buttons. This creates the smooth sliding indicator effect.

3. **Styling:**
   - Rail width: `w-16` (64px)
   - Full height: `h-full`
   - Background: `bg-bg-surface-elevated`
   - Border right: `border-r border-border-default`
   - Buttons: 48x48px, centered, rounded-lg
   - Hover state: `hover:bg-bg-hover`
   - Active indicator: `bg-action-primary/10` with rounded corners

4. **Accessibility:**
   - `aria-label` on nav: "Workspace contexts"
   - `aria-label` on each button: "[Context] workspace (Ctrl+[1/2/3])"
   - `aria-current="page"` on active button
   - Buttons are keyboard focusable (native button behavior)

5. **State integration:**
   - Import `useContextStore` from '@v2/stores/contextStore'
   - Get `activeContext` and `setActiveContext` from store
   - onClick calls `setActiveContext(context.id)`

Do NOT implement keyboard shortcuts here - that's Plan 04's responsibility. Just include the shortcut hint in aria-label for discoverability.

Use Tailwind classes with the V2 token system (bg-bg-*, text-text-*, border-border-*, etc.) established in Phase 1.
