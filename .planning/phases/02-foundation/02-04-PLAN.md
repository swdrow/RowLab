<task type="auto">
  <name>Task 1: Create ShellLayout with Keyboard Navigation</name>
  <files>src/v2/layouts/ShellLayout.tsx</files>
  <action>
Create the ShellLayout component that composes ContextRail + WorkspaceSidebar + content area using CSS Grid, with keyboard shortcuts for context switching.

**Layout structure (CSS Grid):**
```
┌────────┬──────────────────────────────────────┐
│        │                                      │
│  Rail  │           Content                    │
│  64px  │           (flex-1)                   │
│        │                                      │
│        ├──────────────┬───────────────────────┤
│        │   Sidebar    │       Main            │
│        │   256px      │       (flex-1)        │
│        │              │                       │
└────────┴──────────────┴───────────────────────┘
```

**Implementation:**

1. **CSS Grid layout:**
   ```tsx
   <div className="v2 h-screen grid grid-cols-[auto_1fr]">
     {/* Rail - always 64px */}
     <aside className="w-16 h-full bg-bg-surface-elevated border-r border-border-default">
       <ContextRail />
     </aside>

     {/* Sidebar + Content nested grid */}
     <div className="grid grid-cols-[auto_1fr] h-full">
       {/* Sidebar - 256px */}
       <aside className="w-64 h-full bg-bg-surface border-r border-border-default overflow-y-auto">
         <WorkspaceSidebar />
       </aside>

       {/* Main content */}
       <main className="h-full overflow-y-auto p-6">
         <Outlet />
       </main>
     </div>
   </div>
   ```

2. **Keyboard shortcuts (Ctrl+1/2/3):**
   - useEffect with keydown listener on window
   - Check `e.ctrlKey || e.metaKey` (support both Windows/Mac)
   - e.key === '1' → setActiveContext('me')
   - e.key === '2' → setActiveContext('coach')
   - e.key === '3' → setActiveContext('admin')
   - Call `e.preventDefault()` to prevent browser shortcuts

3. **Focus management on context switch:**
   - After `setActiveContext`, move focus to first nav item in sidebar
   - Use: `document.querySelector('.workspace-sidebar a')?.focus()`
   - Add `className="workspace-sidebar"` to WorkspaceSidebar's nav element for this query
   - Use small setTimeout (0ms) to wait for React render

4. **Aria live region for context announcements:**
   - Add visually-hidden `<div aria-live="polite">`
   - Update text when context changes: "Switched to [Context] workspace"
   - CSS: `sr-only` class (position: absolute, clip: rect(0,0,0,0), etc.)

**Important:** Do NOT add theme management here - that stays in V2Layout. ShellLayout is purely about the shell structure and navigation.
  </action>

1. TypeScript compiles: `npx tsc --noEmit src/v2/layouts/ShellLayout.tsx`
2. Build passes: `npm run build`
  </verify>

Add ShellLayout as a nested route under V2Layout for the /beta/* routes that need the shell.

**Current structure:**
```jsx

  <Route index element={<BetaHome />} />
</Route>
```

**New structure:**
```jsx
<Route path="/beta" element={<V2Layout />}>
  {/* Routes that use the shell */}
  <Route element={<ShellLayout />}>
    <Route index element={<BetaHome />} />
    {/* Future routes: dashboard, workouts, athletes, etc. */}
  </Route>

  {/* Routes without shell (future: onboarding, etc.) */}
</Route>
```

**Implementation:**
1. Add lazy import for ShellLayout:
   ```jsx
   const ShellLayout = lazy(() => import('@v2/layouts/ShellLayout'));
   ```

2. Wrap BetaHome (and future routes) with ShellLayout:
   ```jsx
   <Route path="/beta" element={<V2Layout />}>
     <Route element={
       <Suspense fallback={<LoadingFallback />}>
         <ShellLayout />
       </Suspense>
     }>
       <Route index element={
         <Suspense fallback={<LoadingFallback />}>
           <BetaHome />
         </Suspense>
       } />
     </Route>
   </Route>
   ```

This creates the route hierarchy: V2Layout (theme + stores) → ShellLayout (rail + sidebar) → Page components
