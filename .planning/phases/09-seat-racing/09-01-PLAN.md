<task type="auto">
  <name>Task 2: Create TanStack Query hooks for seat race sessions</name>
  <files>src/v2/hooks/useSeatRaceSessions.ts</files>
  <action>
Create TanStack Query hooks wrapping all seat race API endpoints.

Query hooks (read operations):
1. `useSeatRaceSessions(options?: { limit?: number; offset?: number })` - List all sessions for active team
   - Query key: `['seatRaceSessions', activeTeamId, options]`
   - Fetch: `GET /api/v1/seat-races?limit=${limit}&offset=${offset}`
   - StaleTime: 5 minutes

2. `useSeatRaceSession(sessionId: string)` - Get single session with full details (pieces, boats, assignments)
   - Query key: `['seatRaceSession', sessionId]`
   - Fetch: `GET /api/v1/seat-races/${sessionId}`
   - Enabled: only when sessionId provided

Mutation hooks (write operations):
3. `useCreateSession()` - Create new session
   - Mutate: `POST /api/v1/seat-races`
   - onSuccess: invalidate ['seatRaceSessions']

4. `useUpdateSession()` - Update session metadata
   - Mutate: `PATCH /api/v1/seat-races/${sessionId}`
   - onSuccess: invalidate ['seatRaceSessions'], ['seatRaceSession', sessionId]

5. `useDeleteSession()` - Delete session
   - Mutate: `DELETE /api/v1/seat-races/${sessionId}`
   - onSuccess: invalidate ['seatRaceSessions']

6. `useAddPiece()` - Add piece to session
   - Mutate: `POST /api/v1/seat-races/${sessionId}/pieces`
   - onSuccess: invalidate ['seatRaceSession', sessionId]

7. `useUpdatePiece()` - Update piece
   - Mutate: `PATCH /api/v1/seat-races/pieces/${pieceId}`
   - onSuccess: invalidate ['seatRaceSession', sessionId]

8. `useDeletePiece()` - Delete piece
   - Mutate: `DELETE /api/v1/seat-races/pieces/${pieceId}`
   - onSuccess: invalidate ['seatRaceSession', sessionId]

9. `useAddBoat()` - Add boat to piece
   - Mutate: `POST /api/v1/seat-races/pieces/${pieceId}/boats`
   - onSuccess: invalidate ['seatRaceSession', sessionId]

10. `useUpdateBoat()` - Update boat (including finishTimeSeconds)
    - Mutate: `PATCH /api/v1/seat-races/boats/${boatId}`
    - onSuccess: invalidate ['seatRaceSession', sessionId]

11. `useDeleteBoat()` - Delete boat
    - Mutate: `DELETE /api/v1/seat-races/boats/${boatId}`
    - onSuccess: invalidate ['seatRaceSession', sessionId]

12. `useSetAssignments()` - Set all assignments for a boat
    - Mutate: `PUT /api/v1/seat-races/boats/${boatId}/assignments`
    - onSuccess: invalidate ['seatRaceSession', sessionId]

13. `useProcessSession()` - Process session and calculate ratings
    - Mutate: `POST /api/v1/seat-races/${sessionId}/process`
    - onSuccess: invalidate ['seatRaceSession', sessionId], ['athleteRatings']

Get activeTeamId from `useSharedStores().teamId` (existing pattern from useErgTests).

Import types from './types/seatRacing'. Follow exact patterns from `useErgTests.ts` and `useLineups.ts`.

  <name>Task 3: Create TanStack Query hooks for athlete ratings</name>
  <files>src/v2/hooks/useAthleteRatings.ts</files>
  <action>
Create TanStack Query hooks for athlete ELO ratings.

Query hooks:
1. `useAthleteRatings(options?: { ratingType?: string; minRaces?: number; side?: string })` - Get ranked athletes
   - Query key: `['athleteRatings', activeTeamId, ratingType, minRaces, side]`
   - Fetch: `GET /api/v1/ratings?teamId=${teamId}&type=${ratingType}&minRaces=${minRaces}`
   - StaleTime: 2 minutes
   - Returns: Array of `RatingWithAthlete` sorted by rating descending
   - Optional client-side filter by side (Port/Starboard) for side-specific rankings

2. `useAthleteRatingHistory(athleteId: string)` - Get rating history for single athlete (for trend chart)
   - Query key: `['athleteRatingHistory', athleteId]`
   - Fetch: `GET /api/v1/ratings/history?athleteId=${athleteId}`
   - Note: If API doesn't exist, this hook can return empty array for Phase 9 MVP

Mutation hooks:
3. `useRecalculateRatings()` - Recalculate all ratings from scratch
   - Mutate: `POST /api/v1/ratings/recalculate`
   - onSuccess: invalidate ['athleteRatings']

**API Route Note:** The `/api/v1/ratings` endpoint does NOT exist yet. Plan 09-08 creates this route in server/routes/ratings.js. This hook structure is created first so the UI components (Plan 09-02) can be developed in parallel. The hook will return errors until Plan 09-08 executes - this is expected.

Alternative: Use the existing `getTeamRankings` service function by creating a simple API route. But for this plan, focus on the hook structure.
